When we got to architecture, we could do something more like this:

Architecture type: N-Tier Architecture

Pros: 
Separation of concerns makes it easier to understand
Layers can be tested in isolation, which speeds up debugging

Cons:
Increased complexity overhead
Higher latency

Best used for:
Enterprise applications with multiple users, complex business logic, and the need for scalability, security, and maintainability.
examples: e-commerce, large scale web services

Should not be used for:
small/simple applications
rapidly changing projects 
low budget endeavors 


Description:
N-tier architecture is an architecture that splits up processes into separate layers,
for example, 
the first layer may be the user-end application
the second layer may recieve API calls from the user application
the third layer may process the user API call, this is where the business logic might start appearing
the fourth layer may send a request to the database layer
the fith layer may house the database, and process requests from the fourth layer

Each layer communicates with the layer on either side of it,
